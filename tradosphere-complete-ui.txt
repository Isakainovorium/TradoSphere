## ðŸ"€ ROUTING PATTERNS & BEHAVIORS

### Dynamic Route Parameters

**Pattern:** `/resource/:dynamicParam`

**Examples:**
```
/feed/signal/:signalId
  Example: /feed/signal/abc123
  Access: All tiers
  Behavior: Opens signal detail (modal on desktop, full page on mobile)

/live/:streamId
  Example: /live/xyz789
  Access: All tiers (view), Elite+ (host)
  Behavior: Loads live stream view

/profile/:username
  Example: /profile/trader_pro
  Access: All tiers
  Behavior: Shows public profile for that user

/buddies/messages/:conversationId
  Example: /buddies/messages/conv456
  Access: All tiers (must be buddies)
  Behavior: Opens conversation thread

/portfolio/trades/:tradeId
  Example: /portfolio/trades/trade789
  Access: All tiers
  Behavior: Shows detailed trade analysis

/competitions/:competitionId
  Example: /competitions/btc-battle-jan
  Access: All tiers
  Behavior: Shows competition details + leaderboard
```

### Query Parameters

**Pattern:** `/route?param=value&param2=value2`

**Common Query Parameters:**
```
/feed?filter=following
  - filter: all | following | top-signals
  - Example: /feed?filter=following

/discover/creators?tier=elite&sort=followers
  - tier: free | grow | elite | gladiator | legend
  - sort: followers | win-rate | subscribers
  - Example: /discover/creators?tier=elite&sort=win-rate

/competitions?status=active&type=1v1
  - status: upcoming | active | completed
  - type: 1v1 | team | ffa
  - Example: /competitions?status=active&type=1v1

/leaderboards?metric=pl&timeframe=30d
  - metric: pl | win-rate | followers
  - timeframe: 7d | 30d | all-time
  - Example: /leaderboards?metric=pl&timeframe=30d

/portfolio/trades?date-from=2025-01-01&date-to=2025-01-31
  - date-from: ISO date
  - date-to: ISO date
  - Example: /portfolio/trades?date-from=2025-01-01
```

### Protected Routes

**Authentication Check:**
```
IF user NOT logged in:
  REDIRECT to /login with ?redirect=[intended-url]
  Example: User tries to visit /portfolio
  → Redirects to /login?redirect=/portfolio
  → After login, redirects back to /portfolio
```

**Tier Access Check:**
```
IF user tier < required tier:
  SHOW upgrade modal/banner on page
  DISABLE locked features with tier badge indicator
  EXAMPLE: Free user visits /learning
  → Shows page with "Upgrade to TS Grow" modal
```

### Special Routes

**Default Routes:**
```
/ (root)
  - IF logged in → REDIRECT to /feed
  - IF NOT logged in → SHOW landing page

/settings (root)
  → REDIRECT to /settings/profile

/creator (root)
  - IF Elite+ → REDIRECT to /creator/dashboard
  - IF NOT Elite+ → REDIRECT to /settings/subscription

/profile (without username)
  → REDIRECT to /profile/:currentUsername
```

**Modal Routes (Desktop only):**
```
/feed/signal/:signalId
  - Desktop: Opens as modal overlay on /feed
  - Mobile: Full page navigation
  - Close modal → Returns to /feed

Create Signal (triggered from button)
  - Desktop: Modal overlay, URL stays on current page
  - Mobile: Full page, URL changes to /create-signal
```

**Route:** `/pricing`  
**Access:** Public  
**Navigation From:** Landing page, header, settings  
**Navigation To:**
- /signup?tier=[selected-tier] (Upgrade CTAs)
- /login (If user wants to login first)

**Behavior:**
- Highlights user's current tier if logged in
- Upgrade buttons include tier parameter
- Comparison table scrolls horizontally on mobile

---

#### 3-5. About, FAQ, Legal Pages
**Routes:** `/about`, `/faq`, `/legal/terms`, `/legal/privacy`, `/legal/disclaimer`  
**Access:** Public  
**Navigation From:** Footer, legal links during signup  
**Navigation To:** Back to previous page

### AUTHENTICATION PAGES

#### 6. Login Page
**Route:** `/login`  
**Query Params:** `?redirect=/intended-url` (optional)  
**Access:** Public  
**Navigation From:** Header login link, protected route redirects  
**Navigation To:**
- /feed (Default after successful login)
- ?redirect URL (If specified)
- /signup (Sign up link)
- /reset-password (Forgot password link)

**Behavior:**
- Remember redirect URL in query
- Show loading spinner during auth
- Error messages shake animation
- OAuth buttons redirect to providers

---

#### 7. Sign Up Page
**Route:** `/signup`  
**Query Params:** `?tier=[tier-name]` (optional - pre-selects tier)  
**Access:** Public  
**Navigation From:** Landing CTAs, login page, pricing page  
**Navigation To:**
- /verify-email (After successful signup)
- /login (Already have account link)

**Behavior:**
- Pre-select tier from query param
- Real-time password strength validation
- Terms checkbox required
- Disable submit until valid

---

#### 8. Email Verification
**Route:** `/verify-email`  
**Access:** Post-signup (no auth yet)  
**Navigation From:** /signup completion  
**Navigation To:**
- /feed (After verification complete)
- /signup (Change email link)

**Behavior:**
- Can resend verification email
- Auto-redirect after verification
- Countdown timer before resend available

---

#### 9. Password Reset
**Route:** `/reset-password`  
**Query Params:** `?token=[reset-token]` (for step 2)  
**Access:** Public  
**Navigation From:** /login forgot password link, email link  
**Navigation To:**
- /login (After successful reset)

**Behavior:**
- Step 1: Enter email, send reset link
- Step 2: Enter new password with token
- Success → Redirect to /login with success message

### ONBOARDING FLOW

#### 10. Welcome Tour
**Route:** Modal overlay (no URL change)  
**Access:** New users on first login  
**Trigger:** Automatically on first visit to /feed  
**Navigation:**
- Overlays on top of /feed, /live, /portfolio
- Changes underlying page without URL change
- Progress saved to localStorage
- Can resume from last stop
- Skip button → Dismisses tour
- Complete → Shows tier-specific upsell modal

**Tour Progression:**
```
Stop 1: /feed highlight → Explain feed
Stop 2: Navigate to /live → Explain streaming
Stop 3: Navigate to /portfolio → Explain tracking
Stop 4: Show upsell modal → Encourage upgrade
```

```
1. Generate canonical URL (full path + query params)

3. Copy to clipboard
4. Show success toast
5. Optional: Open native share sheet (mobile)
```
